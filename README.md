# INFORME: Comparación entre Programación Tradicional y Programación Orientada a Objetos (POO) en Python
# 1. Introducción

En la presente tarea práctica se realizó una comparación entre la Programación Tradicional y la Programación Orientada a Objetos (POO) mediante el desarrollo de un programa en Python que permite calcular el promedio semanal del clima, a partir de las temperaturas ingresadas diariamente.

El objetivo principal es comprender las diferencias entre ambos paradigmas de programación y aplicar sus conceptos fundamentales en un caso práctico.

# 2. Objetivo

Desarrollar habilidades prácticas en Python aplicando:

Programación Tradicional mediante funciones.

Programación Orientada a Objetos mediante clases y métodos.

Ambos enfoques permiten resolver el mismo problema, pero con estructuras y filosofías distintas.

# 3. Desarrollo
3.1 Programación Tradicional

En la programación tradicional, el problema se resolvió utilizando funciones, las cuales permiten organizar el código de forma modular.

Características principales:

Uso de funciones para dividir el programa en partes.

Entrada de datos diarios mediante una función.

Cálculo del promedio semanal mediante otra función.

Flujo de ejecución secuencial.

Este enfoque es sencillo y fácil de entender, ideal para programas pequeños o tareas específicas.

3.2 Programación Orientada a Objetos (POO)

En la solución orientada a objetos se diseñó una clase que representa la información del clima semanal.

Conceptos aplicados:

Encapsulamiento:
Las temperaturas se almacenan en un atributo protegido dentro de la clase, evitando el acceso directo desde fuera.

Herencia:
Se creó una clase hija que hereda de la clase principal para reutilizar funcionalidades.

Polimorfismo:
Se utilizó un método que aprovecha el cálculo del promedio heredado, adaptándolo a una nueva forma de presentación.

Este paradigma permite un código más organizado, reutilizable y fácil de mantener en programas de mayor tamaño.

# 4. Comparación entre ambos enfoques
Programación Tradicional	Programación Orientada a Objetos
Usa funciones	Usa clases y objetos
Más simple	Más estructurada
Ideal para programas pequeños	Ideal para programas grandes
Menor reutilización	Alta reutilización de código
# 5. Conclusión

Ambos paradigmas permiten resolver correctamente el problema del cálculo del promedio semanal del clima.
La programación tradicional resulta más directa y sencilla, mientras que la programación orientada a objetos ofrece una mejor organización del código y facilita su mantenimiento y escalabilidad.

La elección del paradigma dependerá del tamaño y la complejidad del proyecto a desarrollar.
